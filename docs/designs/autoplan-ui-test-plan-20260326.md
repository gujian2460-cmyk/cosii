# UI 层测试计划（/autoplan Eng Phase 3 artifact）

**范围:** 微信小程序界面（非当前仓库 API 代码）  
**关联评审:** `docs/designs/autoplan-review-ui.md`

## 自动化分层

| 层级 | 工具候选 | 覆盖内容 |
|------|----------|----------|
| 组件/页面单测 | miniprogram-sim / 厂商测试框架 | 状态映射、格式化函数、路由参数 |
| 集成 | 微信开发者工具 CI / 自定义脚本 | 关键页面渲染、mock API |
| E2E | miniprogram-automator + 真机/模拟器 | 登录后下单、支付 mock、约妆选档 |

## 路径 → 测试类型（与工程计划 T7–T10 对齐）

| 用户路径 | 必测交互 | 回归重点 |
|----------|----------|----------|
| 首页 → 商品详情 → 下单 | skeleton → 列表 → 详情 CTA | 首屏仅一个主 CTA；错误 toast |
| 妆娘档期满 | shimmer → 空档 → 选档 → 占位倒计时 | `BOOKING_SLOT_CONFLICT` 映射文案 |
| 支付返回 | 轮询 / 处理中 / 成功 / 超时 | `trace_id` 展示；重试次数 UX |
| 订单详情 | 时间线 + 状态标签 | 与 `unified_orders`/域状态一致展示 |
| 消息 → 深链订单 | 点击跳转订单 id | 禁止在消息里复制完整状态机 |

## CRITICAL 与 UI 相关（T11–T17 中）

- **T12 UI 侧:** 并发约妆下前端不得乐观覆盖已失效档期（刷新后冲突提示）。
- **支付:** 展示层不得把「失败」与「处理中」混用；对齐 `PAYMENT_TIMEOUT` UX 表。
