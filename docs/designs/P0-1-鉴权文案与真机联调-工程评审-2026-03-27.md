# P0-1 鉴权文案与真机联调 — 工程评审（2026-03-27）

## 范围

- **文案**：各模块 `routes.ts` 在未解析到 `userId` 时原抛出 `AUTH_UNAUTHORIZED` + `"Missing X-User-Id"`；生产语义实为「未认证」（可能缺 Bearer、或生产已禁 dev 头）。统一为常量 **`AUTH_REQUIRED_MESSAGE`**（`Authentication required`），**错误码不变**（`AUTH_UNAUTHORIZED`），客户端仍以 `errorUxMap` 为主展示。
- **上线**：在清单中显式要求 **真实环境变量**（`WECHAT_*`、`COSII_SESSION_SECRET`、`COSII_TRUST_DEV_HEADER=0` 等）与 **真机一次** `wx.login` → `/v1/auth/wechat-login` → Bearer 业务请求，并留证据。

## Step 0（范围）

| 项 | 结论 |
|----|------|
| 最小改动 | 单一常量 + 路由替换；清单补勾选项 |
| 复用 | 不新增中间件；保持各路由现有 `if (!userId)` 形态 |
| 复杂度 | 1 新文件 + 8 个 `routes.ts` 引用；未超 smell 阈值 |
| 测试 | 现有用例多断言 `code === AUTH_UNAUTHORIZED`，不依赖旧 `message` 字符串 |

## 架构 / 质量

- **DRY**：`src/shared/auth/auth-messages.ts` 一处定义，避免「Missing X-User-Id」与真实失败原因不一致。
- **显式优于隐晦**：API `message` 与内部实现细节（某请求头名）解耦。
