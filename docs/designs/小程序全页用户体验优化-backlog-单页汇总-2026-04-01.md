## 一、全站原则（验收时对照）

1. **双真源**：浅色品牌壳可保留；**支付 / 争议 / 档期 / 核销** 等关键态须对齐 `DESIGN.md` 语义色 + 文案，避免仅靠品牌粉紫表达成败。
2. **状态四件套**：列表/表单尽量统一 **loading → error（可重试）→ 空态 → 有数据**；空与错文案分离。
3. **单主 CTA**：每屏一个最强行动；次要操作用弱样式或文字链。
4. **信任**：Mock / 占位 / 内测能力须有 **「示例」「内测」** 等标识，减少「像真数据」误导。
