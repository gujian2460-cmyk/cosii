## Key Interactions

- **请求层：** 任意使用 `utils/api.request` 的页面 — 成功信封、`code !== OK`、HTTP 4xx/5xx（有/无 JSON）、`fail` 网络错误。
- **开发联调：** `globalData.apiBase` 指向本地 API 时，开发者工具关闭域名校验；真机改为 **https 已备案域名**。
- **鉴权：** `X-User-Id` 仅开发期；生产切换后回归「未登录 / 过期」提示（计划项）。

## Edge Cases

- 后端返回 **200 但 HTML**（代理误配）— 应走失败路径，不崩溃。
- **并发快速切 Tab** 导致多个 in-flight 请求 — 无闪屏错误状态（后续订单页可加 requestId 丢弃旧响应）。
- **超长 `user_title`** — toast 截断或换行策略（微信 `showToast` 字数限制）。

## Critical Paths

- **P0:** `utils/api` 对 HTTP 状态与 envelope 的解析与现有 Fastify 错误响应一致（与 `src/shared/api-envelope` 对齐）。
- **P1:** 首页探活失败时仍可读、可引导启动后端（当前首页已实现文案引导）。

## Automation (future)

- 可选：**微信 miniprogram-automator** 或官方 CLI + 冒烟脚本，CI 成本高，建议在主链 API E2E 绿后再接。
- 短期：**手测清单** + 每次发版前跑 `npm run verify:gates`（后端）+ 小程序连 staging `API_BASE`。
