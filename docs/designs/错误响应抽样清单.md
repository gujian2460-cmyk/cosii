# 错误响应抽样清单（Envelope + UX 映射）

目的：发布前人工或脚本抽样时，对照 **HTTP 状态**、**业务 `code`**、**`error.user_title` / `primary_action` / `retry_policy`** 是否与 `src/shared/errors/ux-map.ts` 一致。

## 成功信封（参考）

所有 2xx 业务成功响应形态见 `src/shared/api-envelope/index.ts`：

- `code`: `"OK"`
- `data`: 业务载荷
- `trace_id`: 字符串，排障主键

## 错误信封（参考）

失败时：

- `code`: 非 `OK` 的业务码（见 `src/shared/errors/codes.ts`）
- `message`: 服务端消息
- `data`: `null`
- `trace_id`: 字符串
- `error`（由 `fail()` 注入，映射自 `errorUxMap`）：
  - `user_title` ← `userTitle`
  - `primary_action` ← `primaryAction`
  - `retry_policy` ← `retryPolicy`
  - `details`: 可选，调试/字段级信息（勿向前端用户展示敏感内容）

## 已映射业务码一览（与 ux-map 对齐）

| `code` | `user_title`（期望关键词） |
|--------|---------------------------|
| `VALIDATION_ERROR` | 信息填写不完整 |
| `AUTH_UNAUTHORIZED` | 需要登录后继续 |
| `RESOURCE_NOT_FOUND` | 内容不存在或已下架 |
| `BOOKING_SLOT_CONFLICT` | 该时段已被占用 |
| `ITEM_UNAVAILABLE` | 商品暂不可下单 |
| `BUSINESS_RULE_VIOLATION` | 当前操作不被允许 |
| `IDEMPOTENCY_CONFLICT` | 重复请求内容不一致 |
| `PAYMENT_TIMEOUT` | 支付结果确认超时 |
| `PAYMENT_WEBHOOK_DUPLICATE` | 支付结果已同步 |
| `DISPUTE_INVALID_STATE` | 当前状态无法发起该操作 |
| `SETTLEMENT_RETRY_EXHAUSTED` | 结算处理中，请稍后查看 |
| `CONTENT_CARD_TARGET_INVALID` | 关联内容已失效 |
| `LOCAL_VERIFICATION_EXPIRED` | 核销码已过期 |
| `LOCAL_VERIFICATION_INVALID` | 核销码不正确 |
| `LOCAL_VERIFICATION_LOCKED` | 尝试次数过多，核销已锁定 |
| `INTERNAL_ERROR` | 服务暂时不可用 |

## 抽样建议（最小集）

1. 故意触发 `RESOURCE_NOT_FOUND`：请求不存在的订单/资源 ID，`error.user_title` 应出现「不存在或已下架」类文案。  
2. 故意触发 `VALIDATION_ERROR`：POST 缺必填字段。  
3. 支付/回调幂等与乱序：**以自动化为准**（`tests/critical-matrix.t11-t17.test.ts`），发布环境再补一次重放演练记录。

## 维护约定

- 新增 `ErrorCode` 时必须同步 `errorUxMap` 与本文表格（或改为从代码生成本文，后续可做 CI 校验）。
