## Affected Pages/Routes

- `/trade/items/new` — 发布商品字段校验、图片上传、分类正确性。
- `/trade/orders` — 下单、支付、收货确认、结算状态显示。
- `/booking/artists/{id}/slots` — 档期展示、并发占位、冲突提示。
- `/booking/orders/{id}` — 定金支付、改期、履约确认、尾款支付。
- `/disputes/new` — 发起仲裁、证据上传、状态更新可见性。
- `/posts/new` + `/posts/{id}` — 发图、挂卡片、卡片失效回退。

## Key Interactions to Verify

- 交易链路：发布商品 -> 下单 -> 支付 -> 发货/面交 -> 确认收货 -> 抽成结算。
- 约妆链路：选档期 -> 占位 -> 定金 -> 履约 -> 尾款 -> 结算。
- 仲裁链路：发起 -> 上传证据 -> 平台裁决 -> 退款/恢复履约。
- 内容转化链路：发图 -> 挂商品/服务卡 -> 点击卡片 -> 创建订单。

## Edge Cases

- 支付回调重复、乱序、延迟。
- 同一档期并发下单导致双约风险。
- 同一妆娘不同时间段重叠预约（非同起止时间）冲突。
- 争议期间退款与结算并发触发。
- 卡片关联目标下架或删除。
- 用户断网/中途退出后重试提交。

## Critical Paths

- P0: `trade order` money path must stay consistent under retries and replay.
- P0: `booking` must guarantee no double-booking under concurrency.
- P0: `booking` must block overlapping intervals (not only exact timestamp duplicates).
- P0: `dispute` must remain auditable with visible user-facing state.
- P0: webhook out-of-order callbacks must never regress terminal order state.
- P1: content conversion must not break if linked targets become invalid.
