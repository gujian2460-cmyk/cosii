## 0.2 TLS 与反代（HTTPS + Nginx → Node）

**你要做的：** 在服务器上安装 Nginx、放入站点配置、用 Certbot 申请证书、确认 443 能访问健康检查。  
**仓库已准备：** [`deploy/nginx-cosii-api.conf.example`](../../deploy/nginx-cosii-api.conf.example)（把 `api.example.com` 改成你的 API 域名）。

### 你在做什么（一句话）

让浏览器和小程序访问 `https://你的API域名` 时，由 **Nginx** 负责 **HTTPS 证书**，再把请求 **转发** 到你本机跑的 Node（默认 `127.0.0.1:3000`）。

### 前置条件

- [0.1](域名与DNS解析小白教程.md) 已完成：`api.你的域名` 的 **A 记录** 已指向这台服务器公网 IP。  
- 腾讯云安全组（或同类防火墙）已放行 **80、443**（22 建议只对你家 IP）。

### 操作概要（Ubuntu 22.04，与 P0 教程一致）

1. SSH 登录服务器。  
2. 安装：

```bash
sudo apt update && sudo apt -y upgrade
sudo apt -y install nginx certbot python3-certbot-nginx
```

3. 把仓库里的 `deploy/nginx-cosii-api.conf.example` 复制到 `/etc/nginx/sites-available/cosii-api`，编辑 `server_name` 为你的 API 域名。  
4. 启用站点并重载：

```bash
sudo ln -sf /etc/nginx/sites-available/cosii-api /etc/nginx/sites-enabled/cosii-api
sudo nginx -t && sudo systemctl reload nginx
```

5. **先保证 Node 已在 3000 监听**（见 0.5 + `deploy/cosii-api.service.example`），否则反代会 502。  
6. 申请证书：

```bash
sudo certbot --nginx -d api.你的域名.com
```

7. **验收（你要做）**

- 浏览器打开 `https://api.你的域名.com/health` 应看到正常响应（或 JSON）。  
- 在你电脑上执行：

```powershell
curl -I https://api.你的域名.com/health
```

期望：`HTTP/2 200`（或 `HTTP/1.1 200`），无证书报错。

更完整的命令合集仍可与 [P0上线阻塞项实操教程（腾讯云服务器+域名）.md](./P0上线阻塞项实操教程（腾讯云服务器+域名）.md) **P0-1** 对照。

## 0.4 小程序「合法域名」

**你要做的：** 登录 [微信公众平台](https://mp.weixin.qq.com/)（小程序），在 **开发管理 → 开发设置 → 服务器域名** 里配置 **request 合法域名**。  
**代码侧**：`miniprogram/app.js` 的 `apiBase` 必须是 **https://你的API域名**（无末尾路径），与证书域名一致。

### 填写规则（易错）

- 只填 **域名**，不要写 `https://`，不要带 `/v1`。  
- 正确示例：`api.cosii.cn`  
- 错误示例：`https://api.cosii.cn`、`api.cosii.cn/v1`

### 验收

- 微信后台保存成功；  
- 小程序 **关闭**「不校验合法域名」后，体验版仍能请求通（见 0.6）。

## 0.5 生产环境变量

**你要做的：** 在服务器上创建环境变量文件（或面板里逐项添加），**值用你自己的密钥**，不要发群、不要提交 Git。  
**仓库已准备：** [`deploy/env.production.example`](../../deploy/env.production.example) + 常驻进程示例 [`deploy/cosii-api.service.example`](../../deploy/cosii-api.service.example)。

### 必配摘要

| 变量 | 生产建议值 |
|------|------------|
| `NODE_ENV` | `production` |
| `COSII_TRUST_DEV_HEADER` | `0` |
| `WECHAT_APPID` / `WECHAT_SECRET` | 与小程序后台一致 |
| `COSII_SESSION_SECRET` | ≥16 位随机串 |
| `DATA_DIR` 或 `DB_PATH` | 见 [go-live-checklist.md](../go-live-checklist.md) |

**逐条人话说明：** [生产环境变量与真机联调（小白版）.md](./生产环境变量与真机联调（小白版）.md)

### systemd 一种做法（概要）

1. `sudo mkdir -p /etc/cosii`  
2. `sudo cp` 你填好的 env 文件到 `/etc/cosii/api.env`（权限 `chmod 600`）。  
3. 复制 `deploy/cosii-api.service.example` → `/etc/systemd/system/cosii-api.service`，改 `WorkingDirectory`、`User` 等。  
4. `sudo systemctl daemon-reload && sudo systemctl enable --now cosii-api`  
5. `curl -I http://127.0.0.1:3000/health` 应为 200。

`deploy/README.md` 有文件索引。

## 0.6 真机一条主链（wx.login → Bearer）

**你要做的：** 用手机扫 **体验版**，完成一次登录，并确认后续请求带 **Authorization: Bearer**。  
**代码：** 仓库已具备 `wechat-login` 与小程序 `api.js` 注入逻辑；你主要改配置与上传版本。

### 检查清单

- [ ] `app.js`：`apiBase` = `https://你的API域名`，`useWeChatSession: true`  
- [ ] `project.config.json`：`appid` 为真实小程序 AppID  
- [ ] 后端已 0.5，`COSII_TRUST_DEV_HEADER=0`  
- [ ] 0.2 + 0.4 已完成  

### 建议证据（脱敏）

- 时间、体验版版本号；  
- 后端日志或截图中出现成功的 `POST /v1/auth/wechat-login`；  
- 随后一条业务请求（如订单列表）成功，且**不要**截图完整 token。

详细步骤与排障：**[生产环境变量与真机联调（小白版）.md](./生产环境变量与真机联调（小白版）.md)** 后半部分。
