# 生产环境变量与真机联调教程（小白版）

面向：**第一次**把 Cosii 后端和小程序接到**真实服务器 / 真机**的同学。读完应能：

1. 知道要在服务器上配哪些变量、各代表什么；  
2. 在微信公众平台里配好域名与密钥；  
3. 用**真手机**走通：`wx.login` → 换 token → 带登录态请求接口。

更完整的上线清单仍以 [go-live-checklist.md](../go-live-checklist.md) 为准；本文只讲**鉴权相关**这条线。

---

## 1. 先搞懂：环境变量是什么？

可以把它想成：**不给代码写死在文件里，而是在启动程序时告诉程序的一组密码和开关**。

- 同样一份代码，在笔记本上跑可以连 `http://127.0.0.1:3000`；  
- 在云服务器上跑时，通过环境变量告诉它：用哪个数据库文件、微信的 AppSecret 是多少、是否允许「开发用假登录头」等。

**注意：** 环境变量里往往是密钥，**不要**提交到 Git，只在服务器面板、Docker、或本机 `.env`（且 `.env` 已在 `.gitignore`）里配置。

---

## 2. 和「本地开发」差在哪里？

| 项目 | 本地（模拟器） | 真实环境（预发/生产） |
|------|----------------|------------------------|
| 小程序请求的 API 地址 | 常是 `http://127.0.0.1:3000`，可勾选「不校验合法域名」 | 必须是 **HTTPS**，且域名要在微信后台登记为 **request 合法域名** |
| 谁代表「当前用户」 | 常靠 `X-User-Id`（开发头）或临时 token | 应靠 **`wx.login` 拿到的 code** 换 **Bearer token**（`Authorization`） |
| 后端是否信任 `X-User-Id` | 开发模式常会信任 | 生产应 **`COSII_TRUST_DEV_HEADER=0`**，**不能**只靠 `X-User-Id` 冒充登录 |

---

## 3. 后端要配哪些变量？（鉴权相关）

下面变量名与仓库代码一致（`src/modules/auth`、`src/shared/auth`）。

### 3.1 必配（生产「只认微信登录 + Bearer」时）

| 变量名 | 作用（人话） | 怎么填 |
|--------|--------------|--------|
| `NODE_ENV` | 告诉 Node 当前是不是生产 | 填 **`production`** |
| `COSII_TRUST_DEV_HEADER` | 是否允许用 `X-User-Id` 当登录（仅开发用） | 生产填 **`0`**（关闭） |
| `WECHAT_APPID` | 你的**小程序 AppID** | 与微信开发者工具 / `project.config.json` 里 **完全一致** |
| `WECHAT_SECRET` | 小程序 **AppSecret**（在微信公众平台查看） | 从后台复制，**不要泄露、不要提交 Git** |
| `COSII_SESSION_SECRET` | 服务端签发 JWT/会话令牌的密钥 | **至少 16 个字符**的随机串（可用密码生成器生成一长串） |

说明：

- 当 `COSII_TRUST_DEV_HEADER=0`（或生产默认严格模式）时，若 **`COSII_SESSION_SECRET` 未设置或短于 16 位**，进程启动会报错——这是**故意**的，防止线上用默认弱密钥。  
- `WECHAT_APPID` / `WECHAT_SECRET` 不配或错误时，`POST /v1/auth/wechat-login` 会失败（错误码如 `WECHAT_NOT_CONFIGURED` / `WECHAT_AUTH_FAILED`）。

### 3.2 常一起配的（运行与数据）

| 变量名 | 作用 |
|--------|------|
| `PORT` | 监听端口，默认 `3000` |
| `HOST` | 监听地址，默认 `0.0.0.0`（对外可访问） |
| `DATA_DIR` 或 `DB_PATH` | SQLite 数据目录或数据库文件路径（见 [go-live-checklist.md](../go-live-checklist.md)） |

### 3.3 其它（与鉴权无关但上线可能要）

同城核销、微信支付等另有变量（如 `LOCAL_CODE_PEPPER`、商户证书等），见 **`docs/go-live-checklist.md`**，本文不展开。

## 4. 在哪儿设置环境变量？（三种常见情况）

不同平台界面不一样，但**本质都是**：在「启动 `node` 跑 Cosii 之前」把这些键值设进环境。

### 4.1 云托管 / PaaS（如 Railway、Render、Fly.io 等）

1. 打开项目 → **Environment** / **Variables**。  
2. 逐条 **Add**：Name = 上表变量名，Value = 你的值。  
3. **保存后重新部署**一次，让新变量生效。

### 4.2 自己的 Linux 服务器 + systemd

在 `service` 文件里用 `Environment=` 或 `EnvironmentFile=` 指向一个只有 root 可读的文件，例如：

```ini
Environment=NODE_ENV=production
Environment=COSII_TRUST_DEV_HEADER=0
Environment=WECHAT_APPID=wx你的AppId
Environment=WECHAT_SECRET=你的Secret
Environment=COSII_SESSION_SECRET=至少16位随机串
```

改完后：`sudo systemctl daemon-reload && sudo systemctl restart 你的服务名`。

### 4.3 本机用命令行临时试跑（仅练习）

PowerShell 示例（**不要把真密钥贴到聊天或截图发公网**）：

```powershell
$env:NODE_ENV="production"
$env:COSII_TRUST_DEV_HEADER="0"
$env:WECHAT_APPID="wx……"
$env:WECHAT_SECRET="……"
$env:COSII_SESSION_SECRET="至少16位随机串"
npm run dev
```

## 5. 小程序端要改什么？

代码在 `miniprogram/app.js` 的 `globalData`（详见 [miniprogram/README.md](../../miniprogram/README.md)）。

### 5.1 指向真实 API（HTTPS）

把 `apiBase` 改成你的**公网 HTTPS 根地址**（**不要**末尾多一个 `/v1`，路径里会自带 `/v1/...`）：

```js
apiBase: "https://api.你的域名.com",
```

本地调试用的 `http://127.0.0.1:3000` 在真机上**不能**当生产用。

### 5.2 打开微信登录流程

把 **`useWeChatSession` 设为 `true`**：

```js
useWeChatSession: true,
```

这样小程序启动时会：`wx.login` → `POST {apiBase}/v1/auth/wechat-login` → 把返回的 `access_token` 存本地；之后 `utils/api.js` 会优先带 **`Authorization: Bearer ...`**，而不是 `X-User-Id`。

### 5.3 小程序 AppID

`project.config.json` 里的 `appid` 必须是你**真实注册的小程序**，且与后端 `WECHAT_APPID` **一致**。

## 6. 微信公众平台里要配什么？

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)（小程序）。  
2. **开发 → 开发管理 → 开发设置**：  
   - 记下 **AppID**、**AppSecret**（后端 `WECHAT_*`）。  
3. **开发 → 开发管理 → 服务器域名**：  
   - **request 合法域名**：填你的 API 域名，例如 `api.你的域名.com`（不要带 `https://`，按微信界面要求填写）。  
   - 必须是 **HTTPS**，证书有效。  
4. 若微信要求**校验文件**放到网站根目录，按提示把文件放到你 **HTTPS 服务**能访问到的路径（与运维/部署方式有关）。

## 7. 真机联调：推荐步骤（一步一步）

目标：在**真实手机**上证明「登录换 token → 带 Bearer 调业务接口」整条链是通的。

### 7.1 准备检查

- [ ] 后端已在公网 HTTPS 可访问，`GET https://你的API/health` 或 `GET /v1/ready` 能返回成功（可用浏览器或 `curl` 试）。  
- [ ] 服务器已设置：`NODE_ENV=production`、`COSII_TRUST_DEV_HEADER=0`、`WECHAT_*`、`COSII_SESSION_SECRET`。  
- [ ] 小程序 `apiBase` 已改为上述 **HTTPS** 根地址，`useWeChatSession: true`。  
- [ ] 微信后台 **request 合法域名** 已包含该 API 域名。

### 7.2 上传体验版（或预览）

1. 用 **微信开发者工具** 打开 `miniprogram` 目录。  
2. **真机调试**或**上传**为体验版，用手机微信扫码打开（不要用「仅模拟器」糊弄过去）。  
3. 在开发者工具里可打开 **真机调试 / 性能监控**，看网络请求（若版本支持）。

### 7.3 在手机上应该发生什么

1. 打开小程序后，客户端会调 **`wx.login`**（用户几乎无感知）。  
2. 应看到（或在调试里看到）请求 **`POST .../v1/auth/wechat-login`**，body 里带 `code`。  
3. 若成功，响应 `code === "OK"`，且 `data.access_token` 为字符串；之后访问订单等接口应带 **`Authorization: Bearer`**。  
4. 若失败：看返回 JSON 里的 **`code` / `message`**（例如未配微信、code 过期、域名未登记）。

### 7.4 常见卡点

| 现象 | 可能原因 |
|------|----------|
| 提示与域名、HTTPS、证书相关 | 未配合法域名、用了 http、证书链不全 |
| `WECHAT_NOT_CONFIGURED` | 服务器没设 `WECHAT_APPID` / `WECHAT_SECRET` |
| `WECHAT_AUTH_FAILED` | AppSecret 错误、code 已用过或过期（再进一次小程序重试） |
| `401` / `AUTH_UNAUTHORIZED` | 未换到 token，或生产仍只靠 `X-User-Id`（检查 `useWeChatSession` 与 `COSII_TRUST_DEV_HEADER`） |
| 服务启动就崩 | `COSII_SESSION_SECRET` 未设或不足 16 位（在 `COSII_TRUST_DEV_HEADER=0` 时） |

### 7.5 建议留的「证据」（给上线清单用）

- 联调**日期**、小程序**版本号**（体验版号）。  
- 一张**脱敏**截图或文字记录：`wechat-login` 成功、`unified-orders`（或其它需登录接口）**200** 且 body `code: OK`。  
- **不要**在截图里露出完整 `Authorization`、`WECHAT_SECRET`、`COSII_SESSION_SECRET`。

## 9. 小结（背下来这 5 条）

1. 生产：**`NODE_ENV=production` + `COSII_TRUST_DEV_HEADER=0` + 长的 `COSII_SESSION_SECRET` + 正确的 `WECHAT_*`**。  
2. 小程序：**HTTPS 的 `apiBase` + `useWeChatSession: true` + AppID 一致**。  
3. 微信后台：**request 合法域名**指向你的 API。  
4. 真机：**扫码体验版**，确认 `wechat-login` 成功再测业务接口。  
5. 密钥只放在服务器环境变量里，**不要**进 Git、不要发群聊。

若某一步报错，把**错误 JSON 里的 `code` 字段**（不要带密钥）记下来，对照本文 §7.4 或 `src/shared/errors/codes.ts` 排查即可。
