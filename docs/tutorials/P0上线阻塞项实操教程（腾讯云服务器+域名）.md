# P0 上线阻塞项实操教程（腾讯云服务器 + 域名）

适用对象：你已经购买了 **腾讯云 Ubuntu 22.04 服务器 + 域名**，希望把 `P0-1 ~ P0-5` 一次做完。  
目标：从“代码可运行”走到“可控上线”。

---

## 总览（先看这个）

你需要完成 5 件事，顺序如下：

1. `P0-1`：HTTPS + 反向代理  
2. `P0-2`：生产鉴权配置（后端 + 小程序）  
3. `P0-3`：真机联调并留证据  
4. `P0-4`：支付 webhook 验签（代码改造）  
5. `P0-5`：日志与告警链路

---

## P0-1 HTTPS 与反向代理

### 1) 腾讯云控制台

- 安全组开放：`22`（建议仅你的 IP）、`80`、`443`
- 不建议长期开放 `3000` 到公网

### 2) 服务器安装 Nginx + Certbot

```bash
sudo apt update
sudo apt -y upgrade
sudo apt -y install nginx certbot python3-certbot-nginx
```

### 3) 配置 Nginx 反代 Node 服务（示例）

```bash
sudo tee /etc/nginx/sites-available/cosii > /dev/null <<'EOF'
server {
  listen 80;
  server_name api.cosii.cn;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
EOF

sudo ln -sf /etc/nginx/sites-available/cosii /etc/nginx/sites-enabled/cosii
sudo nginx -t
sudo systemctl reload nginx
```

### 4) 申请证书

```bash
sudo certbot --nginx -d api.cosii.cn
```

### 5) 验证

```bash
curl -I https://api.cosii.cn/health
```

期望：返回 `HTTP 200`。

## P0-2 微信生产鉴权链路

## 后端环境变量（必须）

```bash
NODE_ENV=production
COSII_TRUST_DEV_HEADER=0
WECHAT_APPID=wxdc3c7f326dfaf7c2
WECHAT_SECRET=在服务器环境变量中填写（不要写入仓库文档）
COSII_SESSION_SECRET=至少16位强随机字符串
```

说明：

- `COSII_TRUST_DEV_HEADER=0` 会关闭 `X-User-Id` 生产信任
- `COSII_SESSION_SECRET` 不合规会导致严格模式下启动失败
- `WECHAT_SECRET` 属于敏感凭据：只放服务器环境变量/密钥管理，不要提交到 Git

推荐做法（示例）：

```bash
# 仅在服务器会话里设置；不要把明文写进仓库文件
export WECHAT_APPID=wxdc3c7f326dfaf7c2
export WECHAT_SECRET='你的实际Secret'
```

## 小程序配置（必须）

在 `miniprogram/app.js`：

- `apiBase` 改成 HTTPS 域名（`https://api.cosii.cn`）
- `useWeChatSession` 改成 `true`

在 `miniprogram/project.config.json`：

- `appid` 使用真实小程序 AppID（不是 `touristappid`）

## P0-3 真机联调证据

## 微信公众平台配置

- 小程序后台 -> 开发管理 -> 服务器域名
- 配置 request 合法域名：`api.cosii.cn`

## 体验版“所有服务均不可用”快速排障（先做这个）

按顺序执行，任何一步失败都先修这一层，不要跳步。

### A. 先验证后端是否真的在线

在服务器上执行：

```bash
curl -I http://127.0.0.1:3000/health
curl -I https://api.cosii.cn/health
sudo systemctl status nginx --no-pager
sudo systemctl status cosii --no-pager
```

期望：两个 `health` 都是 `200`，`nginx` 和 `cosii` 都是 `active (running)`。

### B. 验证域名和证书

在你本地电脑执行：

```bash
nslookup api.cosii.cn
curl -I https://api.cosii.cn/health
```

期望：域名解析到你的服务器 IP；HTTPS 返回 `200`，无证书错误。

### C. 验证小程序后台“合法域名”

在微信公众平台确认：

- request 合法域名必须是：`api.cosii.cn`（不要写 `https://`，不要带路径）
- 若有多个环境，请确认体验版用的是同一 AppID 对应的配置

### D. 验证小程序包内配置

确认体验版上传前，代码里是：

- `apiBase: "https://api.cosii.cn"`
- `useWeChatSession: true`
- `project.config.json` 的 `appid` 为真实 AppID（`wxdc3c7f326dfaf7c2`）

---

### 常见报错 -> 对应修复

| 现象 | 常见原因 | 修复 |
|------|----------|------|
| `request:fail` 全部失败 | 合法域名没配/配错 | 微信后台重新配置 `api.cosii.cn`，重新上传体验版 |
| HTTPS 失败或握手失败 | 证书未生效/域名解析错 | 重新检查 DNS 与 Certbot/Nginx 配置 |
| 只有登录失败，其他也不可用 | `WECHAT_*` 未配置或不匹配 | 核对 `WECHAT_APPID`、`WECHAT_SECRET` 与当前 AppID |
| 后端接口 502/504 | Nginx 到 Node 反代不通 | 检查 `proxy_pass http://127.0.0.1:3000` 与 `cosii` 进程 |
| 中国大陆网络下始终不可用 | ICP 备案未完成（大陆机房） | 完成备案或临时切非大陆合规方案 |

## 真机联调步骤

1. 上传体验版，手机扫码进入小程序
2. 打开后触发 `wx.login`
3. 观察后端收到 `POST /v1/auth/wechat-login`
4. 再访问业务接口，确认带 `Authorization: Bearer ...`

## 需要保留的证据（脱敏）

- 联调日期时间
- 小程序版本号（体验版版本）
- 一张请求成功截图（或日志）：
  - `POST /v1/auth/wechat-login` 成功
  - 一个业务接口成功（例如 `/v1/me/unified-orders`）
- 注意：不要暴露完整 token、secret

## P0-4 支付 webhook 真实性校验（工程任务）

当前 `src/modules/payment/routes.ts` 的 webhook 仅做 body 校验，不做签名验真。  
这一步是代码改造任务，建议拆成一个独立小需求：

### 建议实现最小集

1. 读取微信回调签名相关头（时间戳、随机串、签名）
2. 用微信平台证书公钥验签（拒绝未通过请求）
3. 验签失败返回 `401/403`
4. 验签成功后再进入现有 `processWechatWebhook`

### 测试最小集

- 增加单测：
  - 签名缺失 -> 拒绝
  - 签名错误 -> 拒绝
  - 验签通过 -> 进入既有幂等流程

## P0-5 生产可观测性和响应链路

### 日志

- 启用 `STRUCTURED_ACCESS_LOG=1`
- 验证日志包含：`trace_id`、`path`、`status_code`
- 在日志平台可以按 `trace_id` 搜索

### 告警（至少三条）

1. webhook 连续失败
2. 结算任务重试耗尽/死信
3. 对账异常（金额不一致）

### 值班链路

- 每条告警要有“谁响应、多久响应、升级给谁”
- 写入运行手册并演练一次
