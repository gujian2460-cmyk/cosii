# 域名与 DNS 解析小白教程（给「轨道 0」第 0.1 步）

你**必须自己做**的部分主要是：**买域名**、在域名控制台里**加一条解析**，让例如 `api.你的域名.com` 指向你的**云服务器公网 IP**。代码和服务器上的程序可以由开发完成；但域名和 DNS 是**你的账号、你的钱、你的控制台**，别人代劳不了。

读完本篇，你能对照执行计划里的 **0.1 域名与 DNS** 打勾。

---

## 1. 三个词先搞懂

| 词 | 白话 |
|----|------|
| **域名** | 别人在浏览器或小程序里访问的一串名字，例如 `cosii.cn`、`api.cosii.cn`。像「门牌」，好记。 |
| **服务器 IP** | 云厂商给你的机器地址，一长串数字（IPv4），例如 `123.45.67.89`。像「经纬度」，机器认这个。 |
| **DNS / 解析** | 告诉全网：「当有人访问 `api.xxx.com` 时，请把他带到 IP `123.45.67.89`」。**A 记录**就是「域名 → IPv4」最常见的一种。 |

你要做的核心一件事：**给 API 子域名加一条 A 记录，指到你的服务器公网 IP**。

---

## 2. 你需要准备什么

1. 已购买 **云服务器**（例如腾讯云），并知道它的 **公网 IPv4**（控制台「云服务器」→ 实例详情里能看到）。  
2. 已购买 **域名**（可在腾讯云、阿里云等注册；不一定和服务器同一家）。  
3. 决定 **API 子域名**，例如：`api.你的根域名.com`（与仓库教程示例 `api.cosii.cn` 同结构即可）。

---

## 3. 买域名（若还没有）

1. 打开域名注册商网站（如腾讯云「域名注册」）。  
2. 搜索想要的根域名（如 `something.com` / `something.cn`），加入购物车、付款。  
3. 实名认证按注册商提示完成（国内注册商通常需要）。

**注意：** 根域名和子域名关系：`cosii.cn` 是根；`api.cosii.cn` 是根下面的**子域名**，解析在「DNS 解析」里单独加。

---

## 4. 在腾讯云 DNS 里添加 A 记录（示例）

以下以 **腾讯云 DNSPod** 为例（其他厂商界面文字略有不同，逻辑一样）。

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/) → 搜索 **DNS 解析 DNSPod**（或「我的域名」）。  
2. 找到你买的域名，点 **解析**。  
3. 点 **添加记录**，填写：

   | 字段 | 填什么 |
   |------|--------|
   | **主机记录** | 若 API 地址是 `api.xxx.com`，填 **`api`**（只填前缀，不要写整段域名）。若用根域名本身当 API（不推荐），填 `@`。 |
   | **记录类型** | **A** |
   | **记录值** | 你的服务器 **公网 IPv4**（如 `123.45.67.89`） |
   | **TTL** | 默认 600 或 300 即可 |

4. 保存。

**结果：** 全网逐步生效后，访问 `api.你的域名.com` 会解析到你的服务器 IP。

---

## 5. 怎么知道解析是否生效？

在你自己的 **Windows 电脑**上打开 **PowerShell** 或 **cmd**，执行：

```powershell
nslookup api.你的域名.com
```

看返回里是否出现你填的 **服务器 IP**。若仍是旧地址或报错，等几分钟～几小时再试（各地 DNS 缓存不同）。

有 `dig` 的环境可执行：

```bash
dig +short api.你的域名.com
```

应输出一行 IPv4。

---

## 6. 常见翻车点

| 现象 | 可能原因 |
|------|----------|
| 解析不生效 | 主机记录写错（多写了 `.com`）、或改错域名 |
| 指向旧 IP | 你换了服务器但没改 A 记录 |
| 小程序仍报 request 失败 | **仅完成 DNS 不够**，还要：**HTTPS 证书 + Nginx 反代 + 微信后台配置合法域名** — 见 [`P0上线阻塞项实操教程（腾讯云服务器+域名）.md`](./P0上线阻塞项实操教程（腾讯云服务器+域名）.md) 从 **P0-1** 继续 |
